# Requirements Archive: v1.1 Source Citation Reports

**Archived:** 2026-02-15
**Core Value:** Every answer in the compliance document must be traceable back to its source.

## v1 Requirements — All Complete

### PDF Infrastructure

- [x] **INFRA-01**: PDF generator uses ReportLab with Unicode-compatible font (DejaVu Sans) to prevent encoding crashes on special characters
- [x] **INFRA-02**: PDF generated in-memory using BytesIO buffer (no intermediate filesystem writes in generator)
- [x] **INFRA-03**: Auto-wrapping table layout handles variable-length text (quotes 1000+ chars) without overflow or data loss
- [x] **INFRA-04**: Multi-page PDFs repeat header rows on each page for readability

### Citation Data Model

- [x] **DATA-01**: Citation JSON schema defines per-field structure: question_id, question_text, answer, source_quote, source_section, confidence, reasoning
- [x] **DATA-02**: Confidence levels supported: DIRECT, INFERRED, DEFAULT, NOT FOUND
- [x] **DATA-03**: Schema validation rejects malformed citation JSON with clear error messages (nulls, wrong types, missing required fields)

### MCP Tool

- [x] **TOOL-01**: New `generate_source_report` MCP tool accepts `source_citations_json` string parameter
- [x] **TOOL-02**: Tool parses JSON, validates schema, calls PDF generator, saves to DATA_DIR
- [x] **TOOL-03**: Tool returns TextContent with download link + EmbeddedResource with base64 PDF (mirrors `generate_compliance_doc` pattern)
- [x] **TOOL-04**: Generated PDF filename includes model name and `_sources_` suffix with UUID

### PDF Report Content

- [x] **CITE-01**: Each citation entry displays question ID, question text, and answer
- [x] **CITE-02**: DIRECT citations show verbatim source quote with section attribution
- [x] **CITE-03**: INFERRED citations show related quote(s) and reasoning chain explaining how the answer was derived
- [x] **CITE-04**: DEFAULT citations explain why a standard/assumed value was used
- [x] **CITE-05**: NOT FOUND citations note what information was searched for
- [x] **CITE-06**: Confidence level displayed with visual encoding (color + text label) for each citation
- [x] **CITE-07**: Executive summary section shows confidence breakdown (count per level), total questions, sources consulted
- [x] **CITE-08**: Cross-references use question IDs from questions.json to link back to compliance form
- [x] **CITE-09**: Report includes generation timestamp and model card identifier

### File Management

- [x] **FILE-01**: PDF files stored in DATA_DIR alongside DOCX files
- [x] **FILE-02**: Existing cleanup thread extended to delete `.pdf` files with same 24-hour TTL
- [x] **FILE-03**: Existing `/download/{filename}` route serves PDF files with correct Content-Type

### Context & Workflow

- [x] **FLOW-01**: context.md updated to instruct Claude to track source citations during compliance form generation
- [x] **FLOW-02**: Claude can call `generate_source_report` independently after `generate_compliance_doc`

## Out of Scope (Documented)

| Feature | Reason |
|---------|--------|
| Modifying existing DOCX template | Not needed for source tracking |
| Embedding citations in DOCX | Separate PDF is cleaner |
| Real-time validation during form fill | Source report is post-generation |
| Automatic confidence boosting | Creates audit risk |
| PDF/A archival format | Deferred — verify requirements first |
