# Phase 07 Plan 01: Implement Discovery Engine

**Goal**: Enhance `fetch_hf_model_card` to identify and list all potentially relevant technical documents without fetching them automatically.

## Proposed Changes

### 1. `server.py`
- Add `list_repo_files` import from `huggingface_hub`.
- Implement `discover_relevant_links(text: str, repo_id: str) -> list[dict]` helper:
    - Extract Markdown links with context.
    - Extract HTML links with context.
    - Parse BibTeX blocks for arXiv/URLs.
    - List repo files and include PDFs.
- Refactor `fetch_hf_model_card`:
    - Remove automatic enrichment logic (following links).
    - Call `discover_relevant_links`.
    - Append a structured "DISCOVERED DOCUMENTS" JSON block to the response.
- Update `extract_and_enrich_model_card` (or rename/replace it) to focus only on discovery.

### 2. Tests
- Create `tests/test_link_discovery.py`.
- Test discovery logic with:
    - Plain Markdown links.
    - HTML links.
    - BibTeX entries.
    - Repo file lists (mocked).

## Implementation Steps

### Task 1: Research and Prototyping
1. Verify `list_repo_files` behavior.
2. Develop regex for BibTeX URL/arXiv extraction.

### Task 2: Implement Helper Logic
1. Add `discover_relevant_links` to `server.py`.
2. Add context capture logic (prefix/suffix extraction).

### Task 3: Update Tool Response
1. Modify `fetch_hf_model_card` to use the new discovery logic.
2. Format the output as a clear JSON block for the LLM.

### Task 4: Add Tests
1. Implement `tests/test_link_discovery.py`.
2. Run tests.

## Success Criteria
- [ ] `fetch_hf_model_card` returns a list of links with context.
- [ ] Technical reports in the repo files are discovered.
- [ ] arXiv links inside BibTeX citations are discovered.
- [ ] Automatic fetching of external docs is disabled (prepared for Phase 8).
