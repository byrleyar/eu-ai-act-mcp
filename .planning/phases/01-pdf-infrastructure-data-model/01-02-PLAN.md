**Step 2: Create `pdf_generator.py` at project root.**

The module should contain:

1. **Font registration at module load** (top of file, after imports):
   ```python
   import os
   from reportlab.pdfbase import pdfmetrics
   from reportlab.pdfbase.ttfonts import TTFont

   FONT_DIR = os.path.join(os.path.dirname(os.path.abspath(__file__)), 'fonts')
   pdfmetrics.registerFont(TTFont('DejaVuSans', os.path.join(FONT_DIR, 'DejaVuSans.ttf')))
   ```

2. **Helper function `_escape_xml(text: str) -> str`** that escapes `&`, `<`, `>` characters in user-provided text before embedding in Paragraph markup. This prevents ReportLab's XML parser from choking on user data containing these characters. Use `html.escape()` from stdlib.

3. **`generate_source_report_pdf(output_stream: BytesIO, citations: list[dict]) -> None`** function:
   - Accept BytesIO output_stream and list of citation dicts (pre-validated by citation_schema)
   - Create `SimpleDocTemplate(output_stream, pagesize=letter, topMargin=0.5*inch, bottomMargin=0.5*inch, leftMargin=0.5*inch, rightMargin=0.5*inch)`
   - Define ParagraphStyles using DejaVuSans font:
     - 'CellText': fontName='DejaVuSans', fontSize=8, leading=10
     - 'HeaderText': fontName='DejaVuSans', fontSize=9, textColor=colors.white
     - 'TitleStyle': fontName='DejaVuSans', fontSize=16, spaceAfter=12
     - 'SubtitleStyle': fontName='DejaVuSans', fontSize=10, spaceAfter=6, textColor=colors.grey
   - Build story (list of flowables):
     a. Title: Paragraph("AI Act Compliance - Source Citation Report", TitleStyle)
     b. Subtitle: Paragraph with generation timestamp (use `datetime.now().strftime('%Y-%m-%d %H:%M')`)
     c. Spacer(1, 0.3*inch)
     d. Table with:
        - Header row: ['#', 'Question', 'Answer', 'Source Quote', 'Section', 'Confidence', 'Reasoning']
        - Data rows: Each citation mapped to Paragraph-wrapped cells using CellText style. Use `_escape_xml()` on ALL user-provided string values before wrapping in Paragraph.
        - colWidths: [0.3*inch, 1.3*inch, 1.5*inch, 2.0*inch, 0.6*inch, 0.7*inch, 1.1*inch] (total ~7.5 inches for letter width minus margins)
        - repeatRows=1 for header repetition on multi-page
        - TableStyle with:
          - Header row: dark grey background, white text, center alignment
          - Data rows: alternating row colors (white and light grey/beige), top-aligned
          - Grid lines: thin (0.5pt) grey grid on all cells
          - Padding: 4pt on all sides
   - Call `doc.build(story)`
   - Do NOT call `output_stream.seek(0)` -- let the caller handle stream position (same pattern as docx_generator.py where caller reads from buffer)

**Important constraints:**
- Pure function: no file I/O, no side effects beyond writing to the provided stream
- All text content must be XML-escaped before Paragraph wrapping
- Use `from citation_schema import Citation` only for type documentation in docstring, not at runtime -- the function accepts `list[dict]` to keep the generator decoupled
- Mirror the simplicity of `docx_generator.py` -- this is infrastructure, not the final feature-rich layout (Phase 3 adds visual hierarchy)

**Test cases:**

1. **test_pdf_generates_valid_output** - Basic test: create BytesIO, call generate_source_report_pdf with 1 citation, seek(0), read with PdfReader, assert at least 1 page, assert 'What is the purpose' appears in page text.

2. **test_pdf_in_memory_no_file_io** - Verify no files are created: snapshot `os.listdir('.')` before and after generation, assert no new files. (Confirms INFRA-02.)

3. **test_pdf_unicode_smart_quotes** - Citation with question_text containing smart quotes ("test"), em-dash (--), and curly apostrophe ('s). Should NOT raise any exception. Verify PDF is valid (PdfReader can open it). (Confirms INFRA-01.)

4. **test_pdf_unicode_accented_characters** - Citation with accented text: "cafe, naive, resume, Zurcher Kantonalbank". Generate PDF, verify valid output.

5. **test_pdf_unicode_non_latin** - Citation with Japanese characters in source_quote. Generate PDF, verify no crash (text extraction may not preserve all characters but generation must succeed).

6. **test_pdf_long_text_wrapping** - Citation with source_quote of 1200+ characters ("Lorem ipsum " * 100). Generate PDF, verify it's valid and has at least 1 page. Assert no crash. (Confirms INFRA-03.)

7. **test_pdf_very_long_text_no_data_loss** - Generate with a known long string, use PdfReader to extract text from all pages, verify key phrases from the long text appear somewhere across pages.

8. **test_pdf_multipage_header_repetition** - Create 30+ citations (enough to force multi-page table). Generate PDF. Assert `len(pdf.pages) >= 2`. Extract text from page 2, verify header text ('Question' or 'Answer' or 'Confidence') appears on page 2 as well as page 1. (Confirms INFRA-04.)

9. **test_pdf_all_confidence_levels** - Create 4 citations, one for each ConfidenceLevel value. Generate PDF. Extract text from all pages. Verify all four confidence level strings appear: 'DIRECT', 'INFERRED', 'DEFAULT', 'NOT FOUND'.

10. **test_pdf_empty_optional_fields** - Citation with answer="", source_quote="", source_section="". Generate PDF without error.

11. **test_pdf_xml_special_characters** - Citation with text containing XML special chars: `&`, `<`, `>`, and `"quotes"`. Verify these don't crash ReportLab's Paragraph XML parser.

12. **test_pdf_has_title** - Generate PDF, extract page 1 text, verify "AI Act Compliance" appears in the title.
