# Phase 08 Plan 01: Implement Targeted Fetch Tool

**Goal**: Implement the `fetch_external_document` MCP tool to allow Claude to retrieve and parse technical documents on demand.

## Proposed Changes

### 1. `server.py`
- Add `fetch_external_document(url: str) -> str` tool:
    - Validate URL (SSRF check, absolute URL check).
    - Handle arXiv URL transformation (`/abs/` -> `/pdf/`).
    - Fetch content using `requests` with a reasonable timeout and User-Agent.
    - Parse PDF using `PdfReader` (limited to first 15 pages).
    - Support HTML fallback if content is not a PDF (extract basic text).
    - Return extracted text with a source header.
- Implement `is_safe_url(url: str) -> bool` helper to prevent SSRF.

### 2. Tests
- Create `tests/test_document_fetch.py`:
    - Test URL safety validator.
    - Test arXiv URL transformation.
    - Test PDF parsing logic (mocking the network response).
    - Test handling of large files (>10MB).

## Implementation Steps

### Task 1: Safety & Helper Logic
1. Implement `is_safe_url` to block private IP ranges and localhost.
2. Implement arXiv URL transformation logic.

### Task 2: Implement Tool
1. Define the `fetch_external_document` tool with the `@mcp.tool()` decorator.
2. Integrate fetch and parse logic (reusing/improving on previous `fetch_extra_info` patterns).
3. Ensure robust error handling for network timeouts or malformed PDFs.

### Task 3: Add Tests
1. Implement unit and integration tests in `tests/test_document_fetch.py`.
2. Run all tests to ensure no regressions in discovery or report generation.

## Success Criteria
- [ ] `fetch_external_document` tool is available via MCP.
- [ ] Tool correctly parses arXiv PDFs and returns text.
- [ ] Tool rejects unsafe or local URLs.
- [ ] Tool respects the 10MB file size limit.
