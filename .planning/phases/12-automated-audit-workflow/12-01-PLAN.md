<task type="auto">
  <name>Task 1: Create AuditService with Pydantic models and LLM audit logic</name>
  <files>audit_service.py, requirements.txt</files>
  <action>
Create audit_service.py with the following components:

1. **Pydantic models** (at module top):
   - `ScoreValue = Literal["1", "1i", "2", "3", "4"]` -- scores are STRINGS, not ints (1i would break int parsing)
   - `ConfidenceValue = Literal["DIRECT", "INFERRED", "DEFAULT", "NOT FOUND"]`
   - `FieldAudit(BaseModel)`: question_id (str), answer (str), score (ScoreValue), score_label (str), source_quote (str), source_section (str), confidence (ConfidenceValue), reasoning (str)
   - `AuditSummary(BaseModel)`: total_fields (int), score_1_count (int), score_1i_count (int), score_2_count (int), score_3_count (int), score_4_count (int)
   - `AuditResult(BaseModel)`: model_id (str), audit_timestamp (str), auditor_model (str), summary (AuditSummary), field_audits (List[FieldAudit])

2. **AUDIT_SYSTEM_PROMPT** constant:
   - Open with "You are a skeptical compliance auditor reviewing EU AI Act compliance documentation."
   - Include the EXACT scoring rubric from automated_testing/rough_audit_prompt.md (1/1i/2/3/4 definitions)
   - Include critical rules: "Assume hallucinations may have occurred", "Score 1 only when you can provide a direct quote", "You MUST return exactly one entry per question_id", "Valid score values are ONLY: 1, 1i, 2, 3, 4"
   - Specify JSON output format: array of objects with fields question_id, answer, score, score_label, source_quote, source_section, confidence, reasoning

3. **build_audit_user_prompt(model_card_text: str, questions: list) -> str**: Constructs user message with MODEL CARD section and QUESTIONS TO ANSWER AND AUDIT section (questions injected as JSON).

4. **AuditService class**:
   - `__init__(self, anthropic_client=None)`: Accept optional client; if None, create `anthropic.Anthropic()` from env. Store as self.client.
   - `audit_model(self, ctx: ProcessingContext) -> AuditResult`:
     a. Read model_card.txt from Path(ctx.output_path) -- raise FileNotFoundError if missing
     b. Load questions.json from project root
     c. Build system + user prompts using the functions above
     d. Call self.client.messages.create() with model="claude-sonnet-4-20250514", max_tokens=16000, system=system_prompt, messages=[{"role": "user", "content": user_prompt}]
     e. Extract JSON from response.content[0].text -- handle case where LLM wraps JSON in markdown code fences (strip ```json and ``` if present)
     f. Parse with json.loads(), validate each entry as FieldAudit
     g. Validate: len(field_audits) must equal len(questions), raise ValueError if mismatch
     h. Build AuditSummary using collections.Counter on scores
     i. Build AuditResult with model_id from ctx, ISO 8601 timestamp, auditor_model="claude-sonnet-4-20250514"
     j. Persist: Write compliance_data.json (dict of question_id -> answer) to ctx.output_path
     k. Persist: Write audit_results.json (full AuditResult.model_dump()) to ctx.output_path
     l. Return AuditResult

5. **Add `anthropic` to requirements.txt**: Append "anthropic" at the end of the file.

Key design notes:
- Import ProcessingContext from compliance_service (reuse Phase 11 pattern)
- Use `from pathlib import Path` for file I/O (same as batch_processor.py)
- The prompt must dynamically inject questions from questions.json, NEVER hardcode question IDs
- Handle JSON extraction robustly: LLMs sometimes wrap output in ```json fences
