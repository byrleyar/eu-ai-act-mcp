# Phase 05 Plan 01: Implement Coverage Enforcement

**Goal**: Update the citation validation logic and MCP tool to ensure every question from `questions.json` is present in the source report.

## Proposed Changes

### 1. `citation_schema.py`
- Add a new function `validate_report_coverage(report: CitationReport, required_ids: list[str]) -> None`.
- This function will check if all `required_ids` are present in `report.citations`.
- If missing, it will raise a `ValueError` with a clear message: `Missing citations for [N] questions: [id1, id2, ...]`.

### 2. `server.py`
- In `generate_source_report`:
    - Load `questions.json`.
    - Extract IDs: `required_ids = [q['id'] for q in questions]`.
    - Call `validate_report_coverage(report, required_ids)` after the initial JSON validation.
    - Ensure the tool returns the validation error as text content if it fails.

### 3. Tests
- Create `tests/test_coverage_enforcement.py`.
- Test `validate_report_coverage` with:
    - Exactly matching IDs (Success).
    - Missing IDs (Failure).
    - Extra IDs (Success - technically okay if the LLM adds more, though unlikely).
- Test the integration in `server.py` (if feasible via pytest).

## Implementation Steps

### Task 1: Update `citation_schema.py`
1. Add `validate_report_coverage` function.
2. Ensure it handles empty `required_ids` gracefully.

### Task 2: Update `server.py`
1. Modify `generate_source_report` to perform coverage check.
2. Update the error handling to include the coverage error message.

### Task 3: Add Tests
1. Implement `tests/test_coverage_enforcement.py`.
2. Run all tests to ensure no regressions.

## Success Criteria
- [ ] `generate_source_report` fails if any question ID from `questions.json` is missing from the input JSON.
- [ ] Error message explicitly lists the missing question IDs.
- [ ] Tool succeeds when all 80+ IDs are present.
- [ ] All existing tests pass.
