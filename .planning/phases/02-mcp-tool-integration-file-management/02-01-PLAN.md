Create the tool function with `@mcp.tool()` decorator, placed after the existing `generate_compliance_doc` tool. The function signature:

```python
@mcp.tool()
def generate_source_report(source_citations_json: str, model_name: str = "model") -> list[types.TextContent | types.EmbeddedResource]:
```

Implementation steps (mirror generate_compliance_doc exactly):

1. **Validate JSON:** Call `validate_citation_json(source_citations_json)` in try/except ValueError. On error, return `[types.TextContent(type="text", text=f"Validation Error: {e}")]`. This satisfies TOOL-01 and TOOL-02.

2. **Generate PDF:** Create `BytesIO()` buffer, call `generate_source_report_pdf(buffer, [c.model_dump() for c in report.citations])` in try/except Exception. On error, return list with TextContent error message.

3. **Build filename:** Sanitize model_name using the EXACT same pattern as generate_compliance_doc:
   - `safe_name = "".join(x for x in model_name if x.isalnum() or x in (' ', '_', '-')).strip().replace(' ', '_')`
   - `short_name = safe_name[:30]`
   - `random_suffix = uuid.uuid4().hex[:6]`
   - `filename = f"{short_name}_sources_{random_suffix}.pdf"`
   This satisfies TOOL-04. Note the `_sources_` infix distinguishes from compliance docs.

4. **Save to DATA_DIR:** Write pdf_bytes to `os.path.join(DATA_DIR, filename)`. This satisfies FILE-01.

5. **Base64 encode:** Use `base64.b64encode(pdf_bytes).decode('utf-8')` — NOT urlsafe_b64encode (causes MCP validation errors per research).

6. **Build download URL:** Copy the EXACT URL construction logic from generate_compliance_doc:
   - Check `RAILWAY_PUBLIC_DOMAIN` or `RAILWAY_STATIC_URL` env vars
   - Prepend `https://` if missing scheme
   - Fallback to server_config.json
   - Construct `full_link = f"{base_url}/download/{filename}"` if base_url else relative path
   This satisfies Railway deployment (success criteria 7).

7. **Return response:** Return list with TextContent (download link with 24-hour notice) and EmbeddedResource (base64 PDF with mimeType "application/pdf" and uri "file:///{filename}"). This satisfies TOOL-03.

The docstring should describe the tool's purpose clearly:
```python
"""
Generate a PDF source citation report from validated JSON.

Validates the citation JSON against the schema, generates a formatted PDF report
showing each question's answer with its source, confidence level, and reasoning,
then returns a download link and the embedded PDF.

Args:
    source_citations_json: JSON string with a "citations" array of citation objects
    model_name: Optional model name for the filename (default: "model")

Returns:
    List containing TextContent (download link) and EmbeddedResource (base64 PDF)
"""
```

IMPORTANT: Always return `list[types.TextContent | types.EmbeddedResource]` even on error paths — never return a bare TextContent without wrapping in a list.
